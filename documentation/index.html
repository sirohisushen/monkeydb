Overview

MonkeyDB is a lightweight, pure Python database engine with basic SQL-like functionality. It is meticulously designed for simplicity, ease of integration, and educational utility, making it an ideal choice for small-scale projects and Python learners.

Features

  • Create tables with specified columns and explicit data types.
  • Insert records into tables with automated unique ID generation.
  • Execute SELECT queries with filtering capabilities based on WHERE clauses.
  • Utilizes JSON files for backend storage enabling human-readable persistence.
  • Employs thread-safe locking mechanisms to ensure concurrency safety.

Installation

Install MonkeyDB with the Python package manager pip:

pip install monkeydb

Usage

Step-by-Step Guide to Use MonkeyDB

  1. Create a dedicated folder to contain your MonkeyDB project files.
  2. Inside the same directory as your root Python script, create a new Python file named monkeyDB.py.
  3. Import the MonkeyDB engine with the statement: 
    from monkeydb import DatabaseEngine
  4. Instantiate the database engine object: 
    db = DatabaseEngine()
  5. Create a new table specifying columns and their data types: 
    db.execute_query("CREATE TABLE users (id INTEGER, name TEXT, age INTEGER)")
  6. Insert data entries into your table: 
    db.execute_query("INSERT INTO users (id, name, age) VALUES (1, 'Alice', 30)")
  7. Retrieve records with a SELECT statement optionally filtered by criteria: 
    results = db.execute_query("SELECT * FROM users WHERE age = 30")
  8. Output the selected records: 
    print(results)

Example Code


# monkeyDB.py
from monkeydb import DatabaseEngine

db = DatabaseEngine()

db.execute_query("CREATE TABLE users (id INTEGER, name TEXT, age INTEGER)")

db.execute_query("INSERT INTO users (id, name, age) VALUES (1, 'Alice', 30)")
db.execute_query("INSERT INTO users (id, name, age) VALUES (2, 'Bob', 25)")

results = db.execute_query("SELECT * FROM users WHERE age = 30")
print(results)

API Reference

StorageEngine

Handles all JSON file interactions and persistence mechanics.

  • create_table_file(table_name: str) → str: Initializes an empty JSON file for a table.
  • write_data(table_name: str, data: List[Dict[str, Any]]): Commits JSON-serializable data to disk.
  • read_data(table_name: str) → List[Dict[str, Any]]: Loads stored JSON data into memory.

QueryParser

Static utility methods that parse SQL-like commands into internal structured formats.

  • parse_create_table(query: str) → Dict[str, Any]: Parses CREATE TABLE queries.
  • parse_insert(query: str) → Dict[str, Any]: Parses INSERT commands.
  • parse_select(query: str) → Dict[str, Any]: Parses SELECT queries, including optional WHERE filters.

DatabaseEngine

Primary engine interface to execute queries and manage database state.

  • execute_query(query: str) → Union[List[Dict], None]: Processes SQL-like queries and returns results.
  • _create_table(parsed_query: Dict[str, Any]): Handles table creation logic internally.
  • _insert(parsed_query: Dict[str, Any]): Inserts records after parsing.
  • _select(parsed_query: Dict[str, Any]): Filters and retrieves data as per query.
